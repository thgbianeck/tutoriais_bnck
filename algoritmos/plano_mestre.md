## Filosofia e Metodologia do Curso

Este curso é estruturado sobre duas bases pedagógicas complementares: a **Metodologia ADDIE** e a **Taxonomia de Bloom**. A **Metodologia ADDIE** organiza o curso em cinco fases — **Análise** (identificar o que o aluno precisa aprender), **Design** (planejar a estrutura e os materiais), **Desenvolvimento** (criar o conteúdo detalhado e prático), **Implementação** (apresentar cada aula com confirmação do aluno) e **Avaliação** (validar a compreensão por meio de desafios e testes automatizados). A **Taxonomia de Bloom** garante que cada aula evolua em complexidade cognitiva: partimos de **Lembrar** e **Compreender** (teoria e analogias), passamos por **Aplicar** e **Analisar** (código prático e exercícios) e chegamos a **Avaliar** e **Criar** (desafios e comparações entre algoritmos). Juntas, essas metodologias garantem que nenhum conceito seja apresentado sem que o anterior tenha sido solidamente construído.

A **Técnica de Feynman** permeia todas as aulas: cada conceito técnico novo é explicado como se o aluno nunca tivesse ouvido falar sobre ele, com analogias do cotidiano que ancoram o entendimento antes de qualquer formalização técnica. O resultado é uma narrativa densa, literária e profundamente conectada à prática.

**Princípio Central de Independência:** cada aula é uma ilha completa de conhecimento. O código de cada aula não depende do código de aulas anteriores. Você pode estudar a Aula 12 sem ter feito a Aula 7. Cada módulo tem seu próprio arquivo Python, seus próprios testes automatizados com **pytest** e seu próprio entregável funcional e testável.

## Nome e Objetivo do Projeto Prático do Curso

**Nome do Projeto:** AlgoLab — Laboratório de Algoritmos em Python

**Descrição:** O **AlgoLab** é o repositório central do curso. Cada aula produz um módulo Python independente, autocontido e testável, que implementa um ou mais algoritmos com código comentado linha a linha. Ao final do curso, o aluno terá um laboratório completo de algoritmos — do mais simples ao mais avançado — com implementações próprias, testes automatizados e documentação clara. O AlgoLab não é um software funcional único, mas uma coleção organizada de algoritmos implementados do zero, que o aluno pode consultar, modificar e reutilizar ao longo de sua vida como desenvolvedor.

## Divisão em Módulos

### Módulo 1 — ESSENCIAL: Fundamentos de Algoritmos e Python
O aluno compreende o que é um algoritmo, como pensar computacionalmente e domina os blocos de construção do Python necessários para implementar qualquer algoritmo. Este módulo é a base de tudo.

### Módulo 2 — PROFICIENTE: Algoritmos de Busca e Ordenação
O aluno implementa e compara os algoritmos clássicos de busca e ordenação, entendendo a lógica por trás de cada um e seus trade-offs de eficiência.

### Módulo 3 — AVANÇADO: Estruturas de Dados e Algoritmos Sobre Elas
O aluno aprende as estruturas de dados fundamentais — Pilhas, Filas, Dicionários, Conjuntos — e os algoritmos que operam sobre elas, incluindo a poderosa técnica da Recursão.

### Módulo 4 — MESTRE: Algoritmos de Grafos, Paradigmas e Complexidade
O aluno conhece algoritmos de grafos, aprende os grandes paradigmas de projeto de algoritmos (Guloso, Divisão e Conquista, Programação Dinâmica) e compreende como medir e comparar a eficiência de qualquer algoritmo com a notação Big O.

## Configuração Inicial do Ambiente (Windows 11 + VS Code + Python 3.13)

### Passo a Passo de Instalação

**1. Verificar a versão do Python instalada**

Abra o **Terminal Integrado** do VS Code com `Ctrl + '` e digite:

~~~text
python --version
~~~

Você deverá ver algo como `Python 3.13.x`. Se não tiver Python instalado, acesse [python.org](https://python.org), baixe o instalador para Windows e marque obrigatoriamente a opção **"Add Python to PATH"** durante a instalação.

**2. Criar a pasta do projeto e o ambiente virtual**

~~~text
cd C:\Users\SeuUsuario\Documents
mkdir algolab
cd algolab
python -m venv .venv
.venv\Scripts\activate
~~~

Após ativar o ambiente virtual, você verá `(.venv)` no início do terminal. Isso confirma que o ambiente está isolado e pronto.

**3. Instalar o pytest para testes automatizados**

~~~text
pip install pytest
~~~

**4. Confirmar a instalação do pytest**

~~~text
pytest --version
~~~

**5. Extensões recomendadas no VS Code**

Abra o VS Code, vá em **Extensões** com `Ctrl + Shift + X` e instale as seguintes extensões: **Python** da Microsoft (suporte completo à linguagem), **Pylance** (análise inteligente de código e autocomplete avançado), **Python Test Explorer** (interface visual para rodar e visualizar testes), **indent-rainbow** (visualização colorida de indentação — essencial em Python) e **Material Icon Theme** (ícones para organização visual das pastas).

**6. Selecionar o interpretador Python no VS Code**

Pressione `Ctrl + Shift + P`, digite **Python: Select Interpreter** e selecione o interpretador dentro da pasta `.venv` do projeto. Ele aparecerá como `.venv\Scripts\python.exe`.

## Como Executar os Testes de Cada Aula

Com o ambiente virtual ativo, navegue até a pasta da aula desejada e execute:

~~~text
pytest testes/ -v
~~~

O flag **-v** (verbose) mostra o resultado de cada teste individualmente, com o nome do teste e o status (PASSED ou FAILED), facilitando muito o aprendizado e a depuração.

## Boas Práticas de Versionamento

Ao longo do curso, cada aula deve ser registrada no repositório com um **commit semântico**, seguindo o padrão **Conventional Commits**. Os prefixos recomendados são: **feat:** para adição de novo algoritmo ou funcionalidade, **fix:** para correção de um código com erro, **docs:** para atualização de README ou log, **test:** para adição ou correção de testes e **chore:** para tarefas de configuração e organização. Exemplos práticos: `feat: implementa busca linear com testes` na Aula 8, `feat: implementa bubble sort e adiciona comparador` na Aula 10 e `docs: atualiza log_estado_projeto aula 15` após cada aula concluída. Ao final de cada módulo, recomenda-se criar uma **tag** de versão para marcar o marco: `v1.0-fundamentos`, `v2.0-busca-ordenacao`, `v3.0-estruturas-dados` e `v4.0-mestre`.
